# Security Policy

## Reporting a Vulnerability
Please do not open public issues for security-sensitive reports.

Instead:
- Open a GitHub Security Advisory for this repo, or
- Contact the maintainer via GitHub DM.

## Hardening
For maintainers, see:
- `docs/security/hardening.md`
- `docs/security/self-audit-checklist.md`

## Scope
Anything that could lead to:
- secret leakage
- unauthorized access
- remote code execution
- unsafe agent behaviors

Thank you for responsible disclosure.
